## Overview

The `useRegion` hook retrieves the user's currently selected region from the [`<GTProvider>` context](__DOCS_PATH__/api/components/gtprovider).

<Callout>
    `useRegion` is a client-side hook and *can only be used in client-side components*.
    Ensure your app is wrapped in a [`<GTProvider>`](__DOCS_PATH__/api/components/gtprovider).
</Callout>

For server-side usage, see [`getRegion`](__DOCS_PATH__/api/helpers/get-region).

## Examples

### Basic usage

```jsx title="RegionDisplay.jsx" copy
'use client';
import { useRegion } from '__PACKAGE_NAME__';

export default function RegionDisplay() {
  const region = useRegion(); // [!code highlight]
  return <p>Current region: {region ?? 'Not set'}</p>;
}
```

---

## Notes

- Returns `undefined` if the user has not selected a region.
- The region can be set using the [`<RegionSelector>`](__DOCS_PATH__/api/components/region-selector) component or [`useRegionSelector`](__DOCS_PATH__/api/helpers/use-region-selector) hook.

## Next steps

- See [`getRegion`](__DOCS_PATH__/api/helpers/get-region) for the server-side equivalent.
