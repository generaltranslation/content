## Overview

The `<LocaleSelector>` component is used to select the user's locale.
It is a client-side component that provides a dropdown to select the locale.

## Reference

### Returns

A component that allows the user to select their locale.

### Props

- **`locales`** (optional): `string[]`
  - An optional list of locales (e.g., `['en', 'es-MX', 'fr']`) to populate the dropdown. If not provided, the list of locales from the `<GTProvider>` context is used.
- **`customMapping`** (optional): `CustomMapping`
  - An optional object to map locale codes to custom display names, emojis, or other properties. Each value can be a string (used as the display name) or an object with properties such as `name` and `emoji`.
  - Example: `{{ 'en-US': 'English (United States)', 'es': { name: 'Español' } }}`
- **`customNames`** (optional, deprecated): `{[locale: string]: string}`
  - An optional object to map locale codes to custom display names. Use `customMapping` instead.

## Examples

### Basic usage

```jsx
import { LocaleSelector } from '__PACKAGE_NAME__';

export default function MyComponent() {
  return <LocaleSelector />;
}
```

### Usage with `customMapping`

```jsx
import { LocaleSelector } from '__PACKAGE_NAME__';

export default function MyComponent() {
  const myCustomMapping = {
    en: 'English',
    es: 'Español',
    'fr-CA': 'Français (Canada)',
  };
  return <LocaleSelector customMapping={myCustomMapping} />;
}
```

---

## Notes

- The `<LocaleSelector>` component allows you to select a different locale for your app.
- The `<LocaleSelector>` component is not available in the server component.
