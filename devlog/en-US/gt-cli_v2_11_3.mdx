## Overview

`derive()` now supports object and array access. Previously, `derive()` could only resolve function calls — now it can resolve values from objects, arrays, and dictionaries, making it easier to work with lookup patterns.

This builds on the [`derive()` in tagged templates](/devlog/gt-react_v10_15_0) release from `gt-react@10.15.0`.

## What's new

### Object access

Use `derive()` with object property lookups. The CLI statically resolves all possible values at extraction time:

```jsx
const labels = {
  dog: "The dog is running.",
  cat: "The cat is running.",
} as const;

export default function Page({ animal }: { animal: string }) {
  return <p>{t(derive(labels[animal]))}</p>;
}
```

Both static (`labels.dog`) and dynamic (`labels[animal]`) access are supported. For dynamic access, the CLI extracts a translation entry for every possible value.

### Array access

Arrays work the same way:

```jsx
const messages = ["Hello", "Goodbye", "Welcome"] as const;

export default function Page({ index }: { index: number }) {
  return <p>{t(derive(messages[index]))}</p>;
}
```

### Nested access

Chain object and array access to any depth:

```jsx
const content = {
  greetings: {
    formal: "Good morning.",
    casual: "Hey!",
  },
} as const;

// derive(content[category][style]) works
```

### Python support

The same patterns work in Python with `t()` — dictionary and list access are both supported. Use `derive()` as the first argument or inside an f-string:

```python
labels = {
    "dog": "The dog is running.",
    "cat": "The cat is running.",
}

# Standalone derive as first arg
t(derive(labels[status_key]))

# derive inside f-string
t(f"{derive(labels[status_key])}")
```
