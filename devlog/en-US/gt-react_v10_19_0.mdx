## Overview

`gt-react/browser` now supports dev hot reload for translations. Previously, browser-side translations via `t()` and `<T>` required a full page reload to see changes — translations were resolved once at module load time and stayed fixed. With this release, new or updated translations are fetched at runtime during development and cached in localStorage across page refreshes.

This brings `gt-react/browser` to parity with the standard `gt-react` translation workflow, where translations update without a full reload.

## How it works

The compiler does the heavy lifting. When `devHotReload` is enabled, the build step injects runtime translate calls into your modules so that translations are fetched ahead of time during development. For JSX content, `<T>` components use Suspense to resolve translations asynchronously on cache misses.

Translations fetched at runtime are persisted to localStorage with a TTL, so they survive page refreshes without re-fetching.

## Configuration

Enable dev hot reload in your `gt.config.json`:

```json
{
  "files": {
    "gt": {
      "parsingFlags": {
        "devHotReload": true
      }
    }
  }
}
```

You can also enable it selectively for strings or JSX:

```json
{
  "files": {
    "gt": {
      "parsingFlags": {
        "devHotReload": {
          "strings": true,
          "jsx": true
        }
      }
    }
  }
}
```

Dev hot reload requires:
- A `loadTranslations` function in your bootstrap config
- A `projectId`
- A `devApiKey`
- Development mode (`import.meta.env.DEV`)
