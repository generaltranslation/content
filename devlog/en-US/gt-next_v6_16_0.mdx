## Overview

`derive()` can now be used inside the `context` parameter of translation functions and components.
This creates separate translation entries for each context variant, enabling context-aware translations without manual duplication.

## Usage

### JSX

Use `derive()` in the `context` prop of `<T>` to generate context-specific translations:

```tsx
import { T } from 'gt-next';

function getFormality() {
  return isBusinessUser ? 'formal' : 'casual';
}

export default function Greeting() {
  return (
    <T context={derive(getFormality())}>
      Hello, how are you?
    </T>
  );
}
```

This creates two translation entries — one for `"formal"` context and one for `"casual"` — allowing the translator to produce different outputs for the same source text.

### Strings

Use `derive()` in the `$context` option:

```tsx
import { useGT, derive } from 'gt-next';

function getFormality() {
  return isBusinessUser ? 'formal' : 'casual';
}

function Component() {
  const gt = useGT();
  return <p>{gt("Hello, how are you?", { $context: derive(getFormality()) })}</p>;
}
```

### Combining with content derivation

You can use `derive()` in both content and context. This multiplies the translation entries:

```tsx
import { derive, gt } from 'gt-next';

function getFormality() {
  return isBusinessUser ? 'formal' : 'casual';
}

function getSubject(gender) {
  return gender === 'male' ? 'boy' : 'girl';
}

function Component({ gender }) {
  const translation = gt(`The ${derive(getSubject(gender))} is here.`, {
    $context: derive(getFormality()),
  });
  return <p>{translation}</p>;
}
// 2 content variants × 2 context variants = 4 translation entries
```
