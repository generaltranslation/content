## Overview

`gt-node` now supports **runtime translation** via the `tx()` function. Previously a stub that threw `"tx() is not implemented"`, `tx()` now translates strings on the fly — no build step required for new content.

This is useful for translating dynamic content like user-generated text, database values, or any string that isn't known at build time.

## Usage

### Basic

```js
import { tx } from 'gt-node';

const translated = await tx('Processing complete');
```

### With variables (ICU format)

```js
const translated = await tx('Hello, {name}!', {
  name: 'María',
  $locale: 'es',
});
```

### Specifying a locale

```js
const status = await tx('Order shipped', { $locale: 'ja' });
```

## How it works

When `tx()` is called:

1. It checks the local translation cache first.
2. On a cache miss, the string is queued for batch translation (50ms window, up to 25 strings per batch).
3. The batch is sent to the GT API, and results are cached with a configurable TTL.

In-flight deduplication ensures the same string isn't translated twice concurrently.
