`formatList` is a standalone utility function from General Translation's Core library that formats an array into a single locale-aware list string. It wraps the built-in `Intl.ListFormat` API, joining items with the correct separators and conjunction or disjunction wording for each locale.

## Overview [#overview]

Import `formatList` directly from `generaltranslation` and call it with an array and an options object. It does not require an API key or a [GT](/docs/platform/core/reference/gt-class/constructor) instance. For instance-based formatting that inherits the instance locale, use the [`formatList`](/docs/platform/core/reference/gt-class-methods/formatting/format-list) method on a `GT` instance instead.

```typescript
import { formatList } from 'generaltranslation';

const list = formatList(['apple', 'banana', 'orange'], { locales: 'en' });
// "apple, banana, and orange"
```

Signature:

```typescript
formatList(
  array: Array<string | number>,
  options?: { locales?: string | string[] } & Intl.ListFormatOptions
): string
```

*Note: `formatList` returns a flat `string`. To keep the original item types (for mixed-type or JSX arrays), use [`formatListToParts`](/docs/platform/core/reference/utility-functions/formatting/format-list-to-parts) instead.*

## How it works [#how-it-works]

- **Underlying API.** Uses the same [`Intl.ListFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/ListFormat) as the GT class method.
- **Locale resolution.** When `locales` is omitted, it falls back to the library default locale, `en`.
- **List type.** `type: 'conjunction'` produces "and" lists, `'disjunction'` produces "or" lists, and `'unit'` produces unit-style lists.

## Parameters [#parameters]

| Parameter | Description | Type | Optional | Default |
| --- | --- | --- | --- | --- |
| [`array`](#array) | The list of items to format. | `Array<string \| number>` | No | — |
| [`options`](#options) | Formatting configuration, including the target locale(s). | `{ locales?: string \| string[] } & Intl.ListFormatOptions` | Yes | `{}` |

### `array` [#array]

**Type** `Array<string | number>` · **Required**

The list of items to join into a single string.

### `options` [#options]

**Type** `{ locales?: string | string[] } & Intl.ListFormatOptions` · **Optional** · **Default** `{}`

Formatting configuration:

| Property | Description | Type | Optional | Default |
| --- | --- | --- | --- | --- |
| `locales` | Locale(s) for formatting. | `string \| string[]` | Yes | `en` |
| `type` | List formatting type. | `'conjunction' \| 'disjunction' \| 'unit'` | Yes | `'conjunction'` |
| `style` | List formatting style. | `'long' \| 'short' \| 'narrow'` | Yes | `'long'` |

## Returns [#returns]

**Type** `string`

The items joined into a single locale-aware list string.

## Examples [#examples]

```typescript
import { formatList } from 'generaltranslation';

// Conjunction (default)
console.log(formatList(['A', 'B', 'C'], { locales: 'en' }));
// Output: "A, B, and C"

// Disjunction
console.log(formatList(['A', 'B', 'C'], { locales: 'en', type: 'disjunction' }));
// Output: "A, B, or C"

// Spanish disjunction
console.log(formatList(['red', 'green', 'blue'], {
  locales: 'es',
  type: 'disjunction',
}));
// Output: "red, green o blue"
```

## Notes [#notes]

- Returns a flat `string`; use [`formatListToParts`](/docs/platform/core/reference/utility-functions/formatting/format-list-to-parts) when you need to preserve non-string item types.
- Always pass an explicit `locales` value for correct, deterministic output.
