title: DataFormat

`DataFormat` specifies the format of translatable content. It appears as the `dataFormat` field of [`EntryMetadata`](/docs/platform/core/reference/types/entry-metadata) and determines how the translation system processes an entry.

## Overview [#overview]

`DataFormat` is a string union of the four supported content formats. Setting it explicitly ensures content is parsed and translated correctly; auto-detection is possible but explicit specification is recommended.

```typescript
type DataFormat = 'JSX' | 'ICU' | 'I18NEXT' | 'STRING';
```

## Values [#values]

| Value | Description | Content type | Use case |
| --- | --- | --- | --- |
| `'JSX'` | JSX/React component format. | [`JsxChildren`](/docs/platform/core/reference/types/jsx-children) | Rich UI components with elements and variables. |
| `'ICU'` | ICU MessageFormat. | `IcuMessage` (string) | Complex formatting with plurals, dates, and numbers. |
| `'I18NEXT'` | i18next message format. | `I18nextMessage` (string) | Simple interpolation and existing i18next projects. |
| `'STRING'` | Plain string format. | `string` | Simple text content. |

### Format characteristics

| Format | Variables | Pluralization | HTML elements | Date/number formatting |
| --- | --- | --- | --- | --- |
| `JSX` | ✅ Rich variables | ✅ Via branching | ✅ Full HTML support | ✅ Via variables |
| `ICU` | ✅ `{variable}` syntax | ✅ Built-in plurals | ❌ Text only | ✅ Built-in formatters |
| `I18NEXT` | ✅ `{{variable}}` syntax | ✅ Via count | ❌ Text only | ✅ Via formatters |
| `STRING` | ❌ | ❌ | ❌ | ❌ |

## Example [#example]

```typescript
const icuEntry: TranslateManyEntry = {
  source: 'Hello {name}',
  metadata: { dataFormat: 'ICU' }
};

const plainEntry: TranslateManyEntry = {
  source: 'Hello, world!',
  metadata: { dataFormat: 'STRING' }
};
```

## Notes [#notes]

- `DataFormat` determines how content is processed by the translation system.
- Specifying the format in [`EntryMetadata`](/docs/platform/core/reference/types/entry-metadata) ensures proper content processing.
- Auto-detection is possible, but explicit specification is recommended.
