## Overview

The `getLocaleFromNativeStore` function reads the persisted locale directly from native storage (or `localStorage` on web) without requiring React context.
This is useful when you need to access the locale imperatively, such as in navigation guards, API clients, or other non-React code.

<Callout type="warn">
    This function reads from the native store directly, so it can temporarily be out of sync with React state while a locale change is in progress.
    For reactive locale access inside components, use [`useLocale`](/docs/react-native/api/helpers/use-locale) instead.
</Callout>

## Reference

```jsx
import { getLocaleFromNativeStore } from 'gt-react-native';

const locale = getLocaleFromNativeStore(key?);
```

### Parameters

<Properties>
    <Property name="key" type="string" optional={true}>
        The storage key to read from. Defaults to `'generaltranslation.locale'`.
        If your [`<GTProvider>`](/docs/react-native/api/components/gtprovider) uses a custom `localeCookieName`, pass that same value here.
    </Property>
</Properties>

### Returns

`string | null` — The persisted locale string (e.g., `'en-US'`), or `null` if no locale has been stored.

---

## Examples

### Basic usage

Read the persisted locale outside of a React component:

```jsx title="api.js" copy
import { getLocaleFromNativeStore } from 'gt-react-native';

function makeApiRequest(endpoint) {
  const locale = getLocaleFromNativeStore(); // [!code highlight]
  return fetch(endpoint, {
    headers: { 'Accept-Language': locale ?? 'en' },
  });
}
```

### With a custom storage key

If your `<GTProvider>` uses a custom `localeCookieName`, pass the same key:

```jsx title="utils.js" copy
import { getLocaleFromNativeStore } from 'gt-react-native';

const locale = getLocaleFromNativeStore('my-app.locale'); // [!code highlight]
```

## Notes

- On native platforms (iOS/Android), this reads from the native key-value store. On web, it reads from `localStorage`.
- This function is synchronous.
- If called during server-side rendering, it returns `null` and logs a warning.
- For reactive locale access inside React components, prefer [`useLocale`](/docs/react-native/api/helpers/use-locale).

## Next steps

- Use [`useLocale`](/docs/react-native/api/helpers/use-locale) for reactive locale access inside components.
