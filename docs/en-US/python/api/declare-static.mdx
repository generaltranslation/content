## Overview

The `declare_static` function marks content as statically analyzable.
It is an identity function used as a marker for static analysis tools.

```python
from generaltranslation import declare_static

content = declare_static("Hello, world!")
# Returns "Hello, world!" unchanged
```

## Reference

### Parameters

| Name | Type | Description |
|---|---|---|
| `content` | `object` | The content to mark as static. |

### Returns

The same `content` value, unchanged. This is a no-op at runtime.

---

## Behavior

`declare_static` is a pass-through function. Its purpose is to signal to static analysis and build tools that the content should be extracted for translation at build time rather than at runtime.
