---
title: static-jsx
description: The <T> component must only have static children
---

## Overview

Ensures that dynamic content inside [`<T>`](/docs/react/api/components/t) components is wrapped in variable components.

This rule flags dynamic content that appears directly inside [`<T>`](/docs/react/api/components/t) components without being wrapped in appropriate variable components like [`<Var>`](/docs/react/api/components/var) or [`<Branch>`](/docs/react/api/components/branch).

## Auto-fix

This rule provides automatic fixes. When you run `eslint --fix`, the linter will:

- **Wrap ternary expressions** in a [`<Branch>`](/docs/react/api/components/branch) component
- **Wrap logical AND expressions** (`&&`) in a [`<Branch>`](/docs/react/api/components/branch) component
- **Wrap other dynamic expressions** in a [`<Var>`](/docs/react/api/components/var) component
- **Add missing imports** for `Branch` or `Var` to your existing GT import declaration

### Ternary expressions

Ternary expressions are automatically converted to `<Branch>` components:

```jsx
// Before fix
<T>{isActive ? "Active" : "Inactive"}</T>

// After fix
<T><Branch branch={isActive} true="Active">Inactive</Branch></T>
```

### Equality comparisons

When the condition is an equality check, the comparison value is used as the prop name:

```jsx
// Before fix
<T>{status === "admin" ? "Administrator" : "User"}</T>

// After fix
<T><Branch branch={status} admin="Administrator">User</Branch></T>
```

### Chained ternaries

Chained ternaries that share the same branch variable are collapsed into a single `<Branch>` with multiple props:

```jsx
// Before fix
<T>{role === "admin" ? "Administrator" : role === "editor" ? "Editor" : "Viewer"}</T>

// After fix
<T><Branch branch={role} admin="Administrator" editor="Editor">Viewer</Branch></T>
```

### Logical AND

Logical AND expressions are converted to self-closing `<Branch>` components:

```jsx
// Before fix
<T>{isOnline && "Online"}</T>

// After fix
<T><Branch branch={!!isOnline} true="Online" /></T>
```

### Other dynamic expressions

All other dynamic expressions are wrapped in `<Var>`:

```jsx
// Before fix
<T>Hello {userName}!</T>

// After fix
<T>Hello <Var>{userName}</Var>!</T>
```

## Examples

### ❌ Incorrect

Dynamic content appears directly in the `<T>` component:

```jsx
<T>Hello {userName}!</T>
```

```jsx
<T>{isActive ? "Active" : "Inactive"}</T>
```

```jsx
<T>{showLabel && "Label"}</T>
```

### ✅ Correct

Dynamic content is wrapped in a variable component:

```jsx
<T>Hello <Var>{userName}</Var>!</T>
```

Conditional content uses a `<Branch>` component:

```jsx
<T><Branch branch={isActive} true="Active">Inactive</Branch></T>
```

Logical AND uses a `<Branch>` component:

```jsx
<T><Branch branch={!!showLabel} true="Label" /></T>
```
