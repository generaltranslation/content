## Getting started

Add translations to your React app with `gt-react`. Setup differs depending on how your app renders, so choose the quickstart that matches your setup:

<Cards>
  <Card title="SPA Quickstart" href="/docs/react/tutorials/quickstart-spa">
    For client-side rendered apps (Vite, Create React App, webpack). Initialize once with `initializeGTSPA()` — no provider needed.
  </Card>
  <Card title="Server-rendered Quickstart" href="/docs/react/tutorials/quickstart-ssr">
    For SSR frameworks (TanStack Start, React Router, custom SSR). Load a translations snapshot on the server and pass it to `<GTProvider>`.
  </Card>
</Cards>

<Callout type='info'>
  **Want automatic setup?** Run `npx gt@latest` to configure everything with the [Setup Wizard](/docs/cli/init).
</Callout>

## How the two setups differ

| | SPA | Server-rendered |
| --- | --- | --- |
| Initialization | `initializeGTSPA()` in your entry point, awaited before render | `initializeGT()` at module scope in your root route |
| Translation loading | In the browser, during initialization | On the server, per request, via `getTranslationsSnapshot(locale)` |
| Provider | None | `<GTProvider locale={locale} translations={translations}>` |
| Locale detection | Automatic (cookie, then browser language) | Your route loader resolves it from the request |

Everything after setup is identical: mark content with [`<T>`](/docs/react/guides/t), translate strings with [`useGT`](/docs/react/guides/strings), and pre-generate production translations with [`npx gt translate`](/docs/cli/translate).

<Callout type='info'>
  **Using a full-stack framework?** [`gt-next`](/docs/next) covers Next.js and [`gt-tanstack-start`](/docs/tanstack-start) covers TanStack Start — both build on `gt-react` and handle the server integration for you.
</Callout>
