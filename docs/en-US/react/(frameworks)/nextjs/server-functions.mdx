`gt-next/server` exposes async, server-only functions for translating content and reading the current request's locale and region. Import them into React Server Components, Route Handlers, and other server code — never into a `'use client'` component.

These functions complement the shared React API. For translating JSX and strings in components, see [Server components](/docs/react/nextjs/server-components); for the client-and-server hooks and components, see the [Reference](/docs/react/reference/config).

*Note: [`getDefaultLocale`](/docs/node/reference/functions/get-default-locale), [`getLocales`](/docs/node/reference/functions/get-locales), and `getLocaleProperties` are exported from the main `gt-next` entry (not `gt-next/server`). See the [Reference](/docs/react/reference/hooks/use-default-locale) for their hook equivalents.*

## `getGT` [#get-gt]

**Returns** `Promise<(content: string, options?: GTTranslationOptions) => string>`

Returns a function that translates strings into the current request's locale. It is the async, server-only counterpart of the [`useGT`](/docs/react/reference/hooks/use-gt) hook — use it in async components where hooks are unavailable. Strings are translated at build time (or on demand in development); interpolated variables are inserted but never translated.

```tsx title="app/page.tsx"
import { getGT } from 'gt-next/server';

export default async function Page() {
  const gt = await getGT();
  return (
    <div>
      <p>{gt('Hello, world!')}</p>
      <p>{gt('Hello, {name}!', { name: 'Alice' })}</p>
    </div>
  );
}
```

## `getTranslations` [#get-translations]

**Returns** `Promise<function>`

The async counterpart of [`useTranslations`](/docs/react/reference/hooks/use-translations). Use it in async components. It returns a function that looks up entries in your dictionary by id. Pass an optional `rootId` to scope all lookups to a nested section of the dictionary.

```tsx title="app/page.tsx"
import { getTranslations } from 'gt-next/server';

export default async function Page() {
  const t = await getTranslations();
  return <p>{t('greeting')}</p>;
}
```

For the dictionary workflow, see [Translating with dictionaries](/docs/react/guides/translating-with-dictionaries).

## `getMessages` [#get-messages]

**Returns** `Promise<function>`

The async counterpart of [`useMessages`](/docs/react/reference/hooks/use-messages). Use it in async components. It returns a resolver for strings that were registered at module scope with [`msg`](/docs/react/reference/functions/msg).

```tsx title="app/page.tsx"
import { getMessages } from 'gt-next/server';
import { msg } from 'gt-next';

const greeting = msg('Hello, world!');

export default async function Page() {
  const m = await getMessages();
  return <p>{m(greeting)}</p>;
}
```

## `tx` [#tx]

**Signature** `tx(message: string, options?: TxOptions): Promise<string>`

Translates a single string at request time, through the General Translation API, and resolves to the translated string. Use `tx` only for content that is not known at build time — build-time translation with [`getGT`](#get-gt) is faster and should be preferred whenever possible. `tx` is server-only.

```tsx title="app/page.tsx"
import { tx } from 'gt-next/server';

export default async function Page({ params }: { params: { note: string } }) {
  const translated = await tx(params.note);
  return <p>{translated}</p>;
}
```

Options are passed as a second argument. All are optional:

| Option | Description | Type |
| --- | --- | --- |
| `$locale` | Target locale. Defaults to the current request locale. | `string` |
| `$context` | Extra context to guide the translation. | `string` |
| `$maxChars` | Maximum number of characters for the translation. | `number` |
| `$format` | Data format for the message. ICU formatting is applied only when set to `'ICU'`. | `string` |
| `[variable]` | Variables to interpolate into the message, as top-level keys. | `string \| number \| boolean \| Date` |

```tsx
const translated = await tx('The price is {price}', {
  $locale: 'es-MX',
  price: 29.99,
});
```

## `Tx` [#tx-component]

**Returns** `Promise<React.ReactNode>`

The JSX counterpart of [`tx`](#tx): an async component that translates its children at request time. Like `tx`, use it only for content not known at build time; prefer the build-time [`<T>`](/docs/react/reference/components/t) component otherwise. `Tx` is server-only.

```tsx title="app/page.tsx"
import { Tx } from 'gt-next/server';

export default function Page() {
  return (
    <Tx context="greeting">
      <p>Hello, world!</p>
    </Tx>
  );
}
```

`Tx` accepts a `context` string and a `locale` override, in addition to its children.

## `getLocale` [#get-locale]

**Returns** `Promise<string>`

Reads the locale for the current request, such as `en-US`. The locale is resolved by the [middleware](/docs/react/nextjs/middleware) and request handling.

```tsx
import { getLocale } from 'gt-next/server';

const locale = await getLocale(); // 'en-US'
```

## `getLocaleDirection` [#get-locale-direction]

**Signature** `getLocaleDirection(locale: string): 'ltr' | 'rtl'` · `getLocaleDirection(): Promise<'ltr' | 'rtl'>`

Returns the text direction for a locale: `'rtl'` for right-to-left languages, otherwise `'ltr'`. Pass a locale to get the direction synchronously, or call it with no argument to get the direction of the current request locale as a promise.

```tsx
import { getLocaleDirection } from 'gt-next/server';

const arabicDir = getLocaleDirection('ar'); // 'rtl'
const currentDir = await getLocaleDirection(); // Promise<'ltr' | 'rtl'>
```

## `getRegion` [#get-region]

**Returns** `Promise<string | undefined>`

Reads the region code for the current request, such as `US` or `CA`, or `undefined` when no region is set.

```tsx
import { getRegion } from 'gt-next/server';

const region = await getRegion(); // 'US' or undefined
```

## `registerLocale` [#register-locale]

**Signature** `registerLocale(locale: string): void`

Sets the locale for the current request context. Use it in Route Handlers and OG image handlers, where Next.js root params are unavailable. Call it at the top of the request handler, before any other `gt-next` function.

```tsx title="app/og/route.tsx"
import { registerLocale, getGT } from 'gt-next/server';

export async function GET() {
  registerLocale('fr');
  const gt = await getGT();
  // gt now translates into 'fr'
}
```

## `resolveCanonicalLocale` [#resolve-canonical-locale]

**Signature** `resolveCanonicalLocale(locale: string): string`

Resolves a locale — including a custom-mapping alias — to its canonical BCP 47 code. Useful when you accept alias codes and need the standardized form.

```tsx
import { resolveCanonicalLocale } from 'gt-next/server';

resolveCanonicalLocale('brand-french'); // 'fr-FR'
```

*Note: `resolveCanonicalLocale` uses the `customMapping` from your [configuration](/docs/react/nextjs/config). Its output depends on how your aliases are defined.*
