The `<Link>` component from `gt-next/link` is a drop-in replacement for Next.js's `<Link>` that prefixes internal hrefs with the active locale. Use it so navigation keeps the visitor on their current locale — a link to `/home` becomes `/en/home` for an English visitor and `/fr/home` for a French one.

It pairs with the [middleware](/docs/react/nextjs/middleware): the middleware routes locale-prefixed URLs, and this `<Link>` generates them.

## Overview [#overview]

Import the default export from `gt-next/link` and use it exactly like `next/link`. Internal paths are localized automatically; you only change the import.

```tsx title="app/nav.tsx"
import Link from 'gt-next/link';

export function Navigation() {
  return <Link href="/home">Home</Link>; // /en/home for an English visitor
}
```

Because it wraps `next/link`, it accepts all the same props (`LinkProps`) and forwards refs.

## How it works [#how-it-works]

- **Internal hrefs are prefixed** with the current locale by default. The active locale comes from the [`useLocale`](/docs/react/reference/hooks/use-locale) context, so `<Link>` must render under a [`<GTProvider>`](/docs/react/nextjs/server-components#provider).
- **External URLs are never prefixed.** Absolute URLs such as `https://example.com` and protocol-relative `//` hrefs pass through unchanged.
- **Object hrefs** are supported: only the `pathname` is localized; `search` and `hash` are preserved.
- **Opting out.** Pass a specific `locale` string to link to another locale, or `locale={false}` to leave the href unchanged.

## The `locale` prop [#locale-prop]

**Type** `string | false | undefined`

Controls locale prefixing:

- **Omitted (default)** — prefix with the current locale.
- **A locale string** — prefix with that locale. Useful for a language switcher.
- **`false`** — do not prefix; the href is used as-is.

```tsx
import Link from 'gt-next/link';

// Current locale (default): /en/home
<Link href="/home">Home</Link>;

// Explicit locale: /fr/home
<Link href="/home" locale="fr">French</Link>;

// No prefix: /legal
<Link href="/legal" locale={false}>Legal</Link>;

// External URL is never prefixed
<Link href="https://example.com">Example</Link>;
```
