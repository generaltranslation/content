---
title: Telemetry
description: Anonymous development telemetry in General Translation libraries
---

General Translation collects limited, anonymous telemetry from `gt-next` when the `withGTConfig` plugin is used during local Next.js development.

This helps us understand how the Next.js SDK is used and prioritize improvements. Participation is optional.

We use telemetry internally to improve General Translation and review it in aggregate.

## What is collected

When a local Next.js dev server starts with `gt-next` enabled, we may collect:

- `gt-next` and Next.js versions
- bundler and compiler mode
- whether the process appears to be running in CI
- which broad `gt-next` features are enabled
- anonymous identifiers used to deduplicate repeated development sessions

## What is not collected

We do not collect GT project IDs, organization IDs, user IDs, API keys, raw git remotes, file paths, usernames, hostnames, package names, locale lists, logs, stack traces, serialized errors, or environment variable values.

The project-level value we send is an anonymous project hash computed locally with a private salt. The salt never leaves your machine, and telemetry is not joined to GT project, organization, or user tables.

## Opt out

You can disable telemetry with:

```sh
NEXT_TELEMETRY_DISABLED=1
GT_TELEMETRY_DISABLED=1
DO_NOT_TRACK=1
```

You can also disable it in `withGTConfig`:

```ts title="next.config.ts"
import { withGTConfig } from 'gt-next/config';

export default withGTConfig(nextConfig, {
  devServerTelemetry: false,
});
```

## Debug mode

To see the exact payload without sending it, set:

```sh
GT_TELEMETRY_DEBUG=1
```

`gt-next` also honors `NEXT_TELEMETRY_DEBUG=1`.

In debug mode, no telemetry request is sent.
