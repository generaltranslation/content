General Translation (GT) is the full-stack localization platform to translate your app, docs, and content into any language. This page explains the product and points you to the fastest quickstart for your setup.

*New to localization? Start with [Key concepts](/docs/overview/key-concepts) for the vocabulary used throughout these docs.*

## What is General Translation? [#what-is]

General Translation connects your code, content, and translations. It combines the full stack in one product:

- **Open-source i18n libraries.** Internationalize your codebase and load translations at runtime. Use `gt-react` for React apps, `gt-next` for Next.js, the `generaltranslation` Core library for any JavaScript runtime, and libraries for Node.js and Python.
- **An AI-native translation platform.** The [Dashboard](/docs/platform/dashboard/get-started) and the [General Translation API](/docs/platform/openapi/quickstart) generate translations, store them, and let you review and guide the output with context and glossaries.
- **Locadex, the AI agent.** [Locadex](/docs/platform/locadex/get-started) internationalizes your code and translates your content continuously, opening pull requests for every change.

GT builds complete understanding of your codebase and product context, so terminology and workflows stay consistent across every piece. You can adopt the whole stack or just the part you need.

*General Translation is developer-first and trusted by teams at Cursor, Ramp, Profound, Cognition, Mintlify, and ClickHouse.*

## Quickstarts [#quickstarts]

Pick the entry point that matches how you want to work. Each quickstart takes you from install to your first translation.

### Automate everything with Locadex

- **Locadex:** the AI agent which connects your codebase, translations, and content sources — the fastest way to run full-stack localization. See [Get started with Locadex](/docs/platform/locadex/get-started).

### Developer libraries and CLI

- **React:** localize React apps with the `gt-react` library. See [React quickstart](/docs/react/get-started/react-quickstart).
- **Node:** localize Node.js services and scripts with General Translation. See [Node quickstart](/docs/node/quickstart).
- **Python:** localize Python apps and services with General Translation. See [Python quickstart](/docs/python/quickstart).
- **Core library:** use the lower-level `generaltranslation` library for translation, formatting, and locale utilities in any JavaScript runtime. See [Core quickstart](/docs/platform/core/quickstart).
- **CLI:** set up internationalization and translate your project from the command line with `gt`. See [CLI quickstart](/docs/cli/quickstart).

### Platform and API

- **Dashboard:** review and edit translations, guide the AI with context, and manage keys and settings. See [Get started with the Dashboard](/docs/platform/dashboard/get-started).
- **OpenAPI:** call the public General Translation API endpoints directly. See [OpenAPI quickstart](/docs/platform/openapi/quickstart).

### Integrations

- **Sanity:** translate content managed in Sanity with the `gt-sanity` plugin. See [Sanity quickstart](/docs/integrations/sanity/quickstart).
- **Storyblok:** translate Storyblok stories from the Dashboard by connecting a space with credentials. See [Storyblok quickstart](/docs/integrations/storyblok/quickstart).

## FAQs [#faqs]

**What is the difference between internationalization, localization, and translation?** Internationalization prepares your codebase to support multiple languages, translation converts content from one language into another, and localization is the whole process of adapting a product for a locale. See [Key concepts](/docs/overview/key-concepts).

**Do I have to use the platform to use the libraries?** No. The General Translation i18n libraries are free, open-source, and can be configured to load translations from any source. See [React quickstart](/docs/react/get-started/react-quickstart).

**Which library should I use?** Use `gt-react` for React apps and `gt-next` for Next.js. Use the [Node](/docs/node/quickstart) or [Python](/docs/python/quickstart) libraries for backend services, and the [Core library](/docs/platform/core/quickstart) for lower-level control in any JavaScript runtime.

**How do AI agents fit in?** Locadex automates localization for you, and you can also give your own AI coding tools access to the docs. See [Use agents](/docs/overview/use-agents).

**What is a locale?** A locale is a language or dialect, identified by a locale code such as `en-US`. See [Key concepts](/docs/overview/key-concepts#locales) and the [Supported locales](/docs/platform/dashboard/reference/supported-locales) list.
