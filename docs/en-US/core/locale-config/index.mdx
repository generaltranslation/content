## Overview

`LocaleConfig` is a client-safe helper class that provides locale metadata, alias resolution, and `Intl` formatting ã without requiring API keys or project credentials.

It wraps all of the locale and formatting methods available on the [`GT`](/docs/core/class) class, but can be used independently on the client side.

```typescript
import { LocaleConfig } from 'generaltranslation';

const localeConfig = new LocaleConfig({
  defaultLocale: 'en',
  locales: ['es', 'fr', 'ja'],
});
```

---

## When to use LocaleConfig

Use `LocaleConfig` when you need locale or formatting utilities on the client side **without** instantiating the full `GT` class (which requires API credentials).

Common use cases:
- Formatting numbers, dates, currencies, and lists for a target locale
- Resolving locale aliases from a custom mapping
- Checking locale validity, direction (LTR/RTL), or relationships
- Determining which locale to use from a set of candidates

## Formatting methods

All formatting methods accept an optional `targetLocale` parameter. The formatting locale fallback order is: explicit `locales` option ã `targetLocale` ã `defaultLocale` ã `'en'`.

### `formatNum(value, targetLocale?, options?)`

Formats a number using [`Intl.NumberFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/NumberFormat).

```typescript
localeConfig.formatNum(1234.5, 'de');
// "1.234,5"
```

### `formatDateTime(value, targetLocale?, options?)`

Formats a date using [`Intl.DateTimeFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/DateTimeFormat).

```typescript
localeConfig.formatDateTime(new Date(), 'ja');
// "2026/4/27"
```

### `formatCurrency(value, currency, targetLocale?, options?)`

Formats a currency value using [`Intl.NumberFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/NumberFormat) with `style: 'currency'`.

```typescript
localeConfig.formatCurrency(9.99, 'EUR', 'fr');
// "9,99 ã˜"
```

### `formatRelativeTime(value, unit, targetLocale?, options?)`

Formats a relative time using [`Intl.RelativeTimeFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/RelativeTimeFormat).

```typescript
localeConfig.formatRelativeTime(-1, 'day', 'es');
// "hace 1 dûÙa"
```

### `formatRelativeTimeFromDate(date, targetLocale?, options?)`

Computes the relative time from a date (compared to `options.baseDate` or now) and formats it.

```typescript
const yesterday = new Date(Date.now() - 86400000);
localeConfig.formatRelativeTimeFromDate(yesterday, 'en');
// "1 day ago"
```

### `formatList(array, targetLocale?, options?)`

Formats an array as a human-readable list using [`Intl.ListFormat`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl/ListFormat).

```typescript
localeConfig.formatList(['apples', 'oranges', 'bananas'], 'en');
// "apples, oranges, and bananas"
```

### `formatListToParts(array, targetLocale?, options?)`

Like `formatList`, but returns an array of parts instead of a string.

### `formatMessage(message, targetLocale?, options?)`

Formats an ICU message string with variables.

```typescript
localeConfig.formatMessage('Hello {name}!', 'en', {
  variables: { name: 'World' },
});
// "Hello World!"
```

### `formatCutoff(value, targetLocale?, options?)`

Truncates a string according to the specified cutoff format options.

## Locale methods

### `getLocaleName(locale)`

Returns the display name of a locale in the default locale's language.

```typescript
localeConfig.getLocaleName('ja');
// "Japanese"
```

### `getLocaleEmoji(locale)`

Returns the flag emoji for a locale.

```typescript
localeConfig.getLocaleEmoji('fr');
// "Þ¨Þñ"
```

### `getLocaleProperties(locale)`

Returns a [`LocaleProperties`](/docs/core/types/locale-properties) object with metadata about a locale.

### `getLocaleDirection(locale)`

Returns `'ltr'` or `'rtl'` for the given locale.

```typescript
localeConfig.getLocaleDirection('ar');
// "rtl"
```

### `isValidLocale(locale)`

Returns whether a locale string is valid (including custom aliases).

### `standardizeLocale(locale)`

Standardizes a locale string to its canonical BCP 47 form.

```typescript
localeConfig.standardizeLocale('en_US');
// "en-US"
```

### `resolveCanonicalLocale(locale)`

Resolves a locale (including custom aliases) to its canonical BCP 47 code.

### `resolveAliasLocale(locale)`

Resolves a canonical locale back to its custom alias, if one exists.

### `isSameDialect(...locales)`

Returns whether all provided locales are the same dialect (e.g., `en-US` and `en-US`).

### `isSameLanguage(...locales)`

Returns whether all provided locales share the same language (e.g., `en-US` and `en-GB`).

### `isSupersetLocale(superLocale, subLocale)`

Returns whether `superLocale` is a superset of `subLocale` (e.g., `en` is a superset of `en-US`).

### `requiresTranslation(targetLocale, sourceLocale?, approvedLocales?)`

Returns whether translation is needed between the source and target locales.

### `determineLocale(locales, approvedLocales?)`

Determines the best matching locale from a list of candidates against the approved locales.

## Accessing from GT

If you already have a `GT` instance, you can access its `LocaleConfig` via the `localeConfig` property:

```typescript
const gt = new GT({ sourceLocale: 'en', locales: ['es', 'fr'] });
const config = gt.localeConfig;

config.getLocaleName('es');
// "Spanish"
```
